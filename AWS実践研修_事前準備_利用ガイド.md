# AWS実践研修 事前準備・利用ガイド（無料利用・課金事故防止版）

## 対象

本研修では以下のAWSサービスを利用します。

- VPC
- EC2
- RDS
- S3
- ALB（Application Load Balancer）
- Systems Manager
- CloudWatch
- CloudFormation
- Terraform

**利用リージョン：** 東京（ap-northeast-1）

> **【重要】2025年7月15日以降に作成したAWSアカウントについて**
>
> 2025年7月15日以降に新規作成したAWSアカウントは、最初から「無料プラン（Free Plan）」になります。
>
> - サインアップ時に **$100**、初期タスクを完了するとさらに **$100**（合計最大 $200）のクレジットが付与される
> - 無料プランの間は、クレジットの範囲内でのみ利用でき、**自分で「有料プランへアップグレード」しない限り課金は発生しない**
> - 無料プランは「**6か月経過**」または「**クレジットを使い切る**」のいずれか早いほうで終了する
> - クレジット残高が50%・25%・10%になった時点、および無料期間の終了15日前・7日前・2日前に通知メールが届く
>
> 2025年7月15日より前に作成した既存アカウントは、従来どおり「12か月無料利用枠＋常時無料枠」が適用されます。
>
> どちらのアカウントでも、本ガイドの予算設定・アラート設定・リソース削除の手順は**課金事故防止の基本動作として有効**です。

## 1. AWSアカウント作成

1. AWS公式サイトへアクセス：https://aws.amazon.com/jp/
2. 「AWSアカウントを作成」を選択
   - 直接サインアップページへ進む場合：https://portal.aws.amazon.com/billing/signup

### 注意事項

- 個人アカウントで作成すること
- 支払い方法（クレジットカード／デビットカード）の登録が必要
  - 本人確認のため少額（1 USD程度）が一時的に保留され、後日返金される
- メールアドレスは個人利用可能なものを使用すること
- 作成途中でブラウザを閉じないこと
- 登録の途中で「Free plan」「Paid plan」の選択画面が表示された場合は、**Free plan（無料プラン）**を選択する

---

## 2. よくあるトラブル

### 「指定されたメールアドレスは既に AWS アカウントに関連付けられています」

**原因**
過去に同じメールアドレスでAWSアカウントを作成済み。

**対応**
新規作成せず、既存アカウントへログインする。

### 「指定されたサインイン情報の AWS アカウントは存在しません」

**原因**
IAMユーザーでログインしようとしている。

**対応**
Root user を選択してログインする。

### 認証コードが届かない

**原因**
アカウント作成途中ではなく既存アカウントである可能性が高い。

**対応**
パスワード再設定を実施する。

---

## 3. Rootユーザーでログイン

1. AWSサインイン画面へアクセス：https://console.aws.amazon.com/
2. 「Root user」を選択
3. 登録メールアドレスを入力
4. パスワードを入力
5. サインイン

> **注意：** IAMユーザーではありません。

---

## 4. MFA（多要素認証）設定【必須】

1. 右上のアカウント名をクリック
2. 「セキュリティ認証情報」を選択
   - 直接アクセスする場合：https://console.aws.amazon.com/iam/home#/security_credentials
3. 「多要素認証（MFA）」を選択
4. 「MFAを割り当て」をクリック

**推奨：** Google Authenticator

**完了条件**
ログイン時に認証コード入力が要求されること。

---

## 5. 課金通知設定【必須】

1. 右上のアカウントメニューから「請求と支払い」を選択
2. 「予算」を選択
   - 直接アクセスする場合：https://console.aws.amazon.com/billing/home#/budgets
3. 「予算を作成」をクリック

> **補足（無料プランのアカウントの場合）**
> 無料プランのアカウントには、クレジット残高に応じた通知（残り50%・25%・10%、無料期間終了15日前・7日前・2日前）がAWSから自動的に届きます。
> ただし、これは「クレジットの消費状況」の通知であり、コスト管理の練習として以下の予算設定も必ず行ってください。

### 予算設定

| 項目 | 設定値 |
|---|---|
| 予算設定 | カスタマイズ |
| 予算タイプ | コスト予算 |
| 予算設定方法 | 固定 |
| 予算額 | 1 USD（※AWS仕様により0 USDは設定不可） |
| 期間 | 毎月 |
| 予算の範囲 | すべて |
| コスト集計基準 | 非ブレンドコスト |

### 通知設定

**通知①**
- 予測コスト
- 超過
- 80%
- 通知先：自身のメールアドレス

**通知②**
- 実績コスト
- 超過
- 100%
- 通知先：自身のメールアドレス

> **注意：** 通知 ≠ 自動停止
> 通知が届いてもAWSは自動停止しません。

## 6. 無料利用枠アラート設定

1. 「請求と支払い」を選択
   - 直接アクセスする場合：https://console.aws.amazon.com/billing/home#/preferences
2. 「Billing preferences（請求設定）」を選択
3. 「Receive AWS Free Tier alerts（AWS無料利用枠アラートを受け取る）」にチェック
4. 「保存」をクリック

> **補足**
> 既定では、無料利用枠の85%に達した時点でメール通知が届くように設定されています。

---

## 7. コストエクスプローラー有効化

1. 「請求と支払い」を選択
2. 「コストエクスプローラー」を選択
   - 直接アクセスする場合：https://console.aws.amazon.com/cost-management/home#/cost-explorer
3. 「有効化」をクリック

> **注意：** 初回は「準備中」となる場合があります。数分～数時間程度で利用可能になります。

## 8. IAMユーザー作成【必須】

1. 検索バーで「IAM」を検索
   - 直接アクセスする場合：https://console.aws.amazon.com/iam/home#/users
2. 「ユーザー」を選択
3. 「ユーザーを作成」をクリック

### 設定内容

| 項目 | 設定値 |
|---|---|
| ユーザー名 | `training-user` |
| 権限 | ポリシーを直接アタッチ → `AdministratorAccess` |

最後に「作成」をクリック。

> **補足**
> AWSは現在、人が利用するアクセス手段として「IAM Identity Center」の利用を推奨していますが、複数アカウント管理を前提とした仕組みのため、個人で1アカウントのみを使う本研修では、シンプルな**IAMユーザー方式**で進めます。

## 9. RootユーザーとIAMユーザーの違い

### Rootユーザー

AWSアカウント作成時に登録したメールアドレスのアカウント。AWSアカウントの最高権限を持つ。

**Rootユーザーでしかできない操作**

- **アカウント管理**
  - AWSアカウント閉鎖
  - Rootメールアドレス変更
  - Rootパスワード変更
- **支払い管理**
  - クレジットカード変更
  - 支払い方法変更
- **セキュリティ管理**
  - Root MFA設定・解除
- **緊急復旧**
  - IAMユーザーを誤削除した場合の復旧
  - IAM権限を誤設定した場合の復旧

### IAMユーザー

AWS内で作成する運用ユーザー。

- 今回作成するユーザー：`training-user`
- 権限：`AdministratorAccess`

**IAMユーザーを使う理由**

AWS公式推奨：「Rootは初期設定のみ、日常運用はIAM」

研修で利用する以下のサービスは全てIAMユーザーで利用可能です。

- VPC
- EC2
- RDS
- S3
- ALB
- Systems Manager
- CloudWatch
- CloudFormation
- Terraform

## 12. Root利用禁止とは

**意味**
Rootユーザーで日常のAWS運用を行わないこと。

### NG例

```text
Rootログイン → EC2作成 → RDS作成 → S3作成
```

### OK例

```text
Rootログイン → MFA設定 → 予算設定 → IAMユーザー作成 → サインアウト
```

以降は **IAMユーザーでログインして作業する**。

## 15. 演習終了時の削除対象

| サービス | 削除対象 |
|---|---|
| EC2 | インスタンス終了（Terminate） |
| RDS | データベース削除（Delete） |
| ALB | ロードバランサー削除 |
| Target Group | 削除 |
| S3 | オブジェクト削除 → バケット削除 |
| CloudFormation | スタック削除（Delete Stack） |
| Terraform | `terraform destroy` 実行 |
| CloudWatch | アラーム削除・ダッシュボード削除・ログ削除 |

---

## 16. 放置した場合の課金リスク

- **EC2：** 起動状態のまま放置 → 時間課金継続
- **RDS：** 作成したまま放置 → 時間課金継続
- **ALB：** 作成したまま放置 → 時間課金継続
- **Multi-AZ：** 有効のまま放置 → プライマリ＋スタンバイの2台分課金
- **Terraform：** 削除忘れ → 作成したAWSリソースが課金継続

## 17. 課金アラートを受信したときの対応【重要】

予算アラートやFree Planのクレジット残量通知が届いたら、以下の手順で対応します。

### Step 1: どのサービスで費用が発生しているか確認する

1. 「請求と支払い」→「Bills（請求書）」で進行中の費用を確認
   - https://console.aws.amazon.com/billing/home#/bills
2. 「コストエクスプローラー」でサービス別のコストを確認
   - https://console.aws.amazon.com/cost-management/home#/cost-explorer

> **注意：** コストの反映には最大24時間程度のタイムラグがあります。アラートを受信した時点では「すでに課金対象のリソースが起動している」前提で対応してください。

### Step 2: サービスごとの停止・削除方法

| サービス | 操作 | 手順 / URL |
|---|---|---|
| EC2 インスタンス | 終了（Terminate） | インスタンス一覧 → 対象を選択 →「インスタンスの状態」→「インスタンスを終了」<br>https://console.aws.amazon.com/ec2/home?region=ap-northeast-1#Instances |
| EBSボリューム | 削除 | EC2画面 →「ボリューム」→ 未使用（available）のボリュームを選択 → 削除 |
| Elastic IP | リリース | EC2画面 →「Elastic IP」→ 未関連付けのアドレスを選択 →「Elastic IPアドレスの解放」 |
| RDS | 削除（Delete） | データベース一覧 → 対象を選択 →「アクション」→「削除」<br>https://console.aws.amazon.com/rds/home?region=ap-northeast-1#databases: |
| ALB | 削除 | EC2画面 →「ロードバランサー」→ 対象を選択 →「アクション」→「削除」 |
| Target Group | 削除 | EC2画面 →「ターゲットグループ」→ 対象を選択 →「アクション」→「削除」 |
| NAT Gateway | 削除 | VPC画面 →「NATゲートウェイ」→ 対象を選択 →「アクション」→「削除」 |
| S3 | オブジェクト削除 → バケット削除 | バケットを開く → 全オブジェクト選択 →「削除」→ バケット自体を削除（バージョニング有効時は全バージョン・削除マーカーも削除） |
| CloudWatch | アラーム／ダッシュボード／ロググループ削除 | 各メニューから対象を選択して削除 |
| CloudFormation | スタック削除（Delete Stack） | スタック選択 →「削除」（関連リソースも一括削除される） |
| Terraform | `terraform destroy` | 作成したリソース一式をまとめて削除 |

> **特に注意（課金が残りやすいもの）**
> - EC2を「停止（Stop）」しただけではEBSボリュームの課金は止まりません。不要であれば終了（Terminate）してください。
> - 未使用のElastic IP・EBSボリューム・NATゲートウェイは、他のリソースを削除した後も単独で課金が継続します。
> - RDSの「最終スナップショットを作成」を有効にすると、スナップショット分のストレージ課金が継続します。研修データが不要な場合はチェックを外してください。

### Step 3: 削除後の確認

1. 全リソース削除後、対象リージョン（東京 / ap-northeast-1）で「実行中のリソースなし」を確認する
2. コストエクスプローラーで翌日以降のコストが減少していることを確認する（即時には反映されません）
3. 不明な点がある場合は、AWSサポート（ベーシックサポートは無料）に問い合わせる
   - https://console.aws.amazon.com/support/home

> **再確認：** アラートは「通知」のみで自動停止ではありません。アラートを受信したら、上記の手順で**自分で**リソースを停止・削除してください。
