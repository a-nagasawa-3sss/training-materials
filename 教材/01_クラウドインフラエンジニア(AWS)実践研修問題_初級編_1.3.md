## 概要

初級編では、以下の内容を実施します。

- VPC、EC2、RDS、S3、ELB を利用して、可用性の高いブログサイト（WordPress）を構築する
- EC2 Instance Connect / Systems Manager を使用した EC2 インスタンスへのコマンド実行を行い、CloudWatch / CloudWatch Logs を利用したモニタリングを行う
- CloudFormation、Terraform を利用して、シンプルな構成のコードによる構築を行う

## 演習の進め方

次ページ以降に掲載されている「タスクの詳細」を元に、ドキュメントやインターネット上の情報を必要に応じて参照しながら構築を行ってください。

> **複数人で共有している場合：** AWS 環境を複数人で共有している場合は、必要に応じて重複しない一意の名称に置き換えて作業を進めてください。

> **手順書について：** 調査を行ったうえで構築をすることが難しかった場合は、別途配布している各タスクの「手順書」を元に構築を行っていただくことが可能です。

### セルフチェック（各タスク完了後）

各タスクの完了後は、別途配布している **「AWS 構築演習チェックシート」** に各タスクの実施結果のセルフチェックと振り返りを記入し、成果物として講師へ提出してください。

セルフチェックでは、以下の選択肢から各タスクを自己評価します。

- [ ] タスクおよびドキュメント等を元に構築を完了することができた
- [ ] 手順書を元に構築を完了することができた
- [ ] 構築を完了することができなかった

> **⚠️ 注意：** AWS を利用する上で無駄な料金が発生しないよう、演習を終える際には必ずサービスを停止してください。また、演習で自身が使用しているリソース以外（案件で使用している検証環境など）には決して触れないようにしてください。

#### 2-3. WordPress を構築する

EC2 Instance Connect を利用して EC2 インスタンスへ接続し、以下のコマンドを実行します。

```bash
# システムパッケージをアップデート
sudo dnf update -y

# WordPress 実行に必要になる PHP とライブラリをインストール
sudo dnf install php php-cli php-common php-mbstring php-xml php-gd php-mysqlnd php-curl php-zip -y

# MySQL レポジトリと GPG キー、クライアントのインストール
sudo dnf install https://dev.mysql.com/get/mysql80-community-release-el9-5.noarch.rpm -y
sudo rpm --import https://repo.mysql.com/RPM-GPG-KEY-mysql-2022
sudo dnf install mysql-community-client -y

# 初期インストールされている Apache の起動・自動起動設定
sudo systemctl enable httpd.service
sudo systemctl start httpd.service

# WordPress をダウンロードして解凍する
wget https://ja.wordpress.org/latest-ja.tar.gz
tar -xzf latest-ja.tar.gz

# Web サーバー上で公開するためのファイルの複製と権限設定
sudo cp -r ~/wordpress/* /var/www/html/
sudo chown apache:apache -R /var/www/html
```

**WordPress のインストール設定：**

EC2 インスタンスのパブリック IP アドレスへ HTTP でアクセスし、以下の情報を入力します。

| 設定項目 | 値 |
|---------|-----|
| テーブル接頭辞 | `wp_` |
| サイトのタイトル | 任意 |
| ユーザー名 | `admin` |
| パスワード | 初期生成されているものを利用（後で利用するため**必ず控えること**） |
| メールアドレス | 自身のメールアドレス |
| 検索エンジンでの表示 | チェックを入れる（検索されないようにする） |

> **※** ここでは、初期構築や動作確認を行いやすくするため、EC2 Instance Connect を利用して接続します。後続のタスクでは、Systems Manager を利用した接続やコマンド実行も確認します。

---

### タスクの要件

#### 3-1. S3 バケットを作成する

| 設定項目 | 値 |
|---------|-----|
| バケット名 | `wordpress-123456789`（ランダム文字列を使って一意にする） |
| リージョン | 東京（`ap-northeast-1`） |
| ACL | 有効 |
| パブリックアクセス | 「パブリックアクセスをすべてブロック」のチェックを**外す** |

---

#### 3-2. IAM ロールを作成し EC2 インスタンスへ割り当てる

| 設定項目 | 値 |
|---------|-----|
| ロール名 | `ec2-wordpress-role` |
| IAM ポリシー | `AmazonS3FullAccess` |
| 割り当て先 | `WordPress 01` インスタンス |

---

#### 3-3. S3 を利用するためのプラグインを WordPress へ追加する

1. WordPress 管理者ページへアクセスする
   ```
   http://<EC2 のパブリック IP アドレス>/wp-login.php
   ```

2. ログイン情報を入力する
   - ユーザー名：`admin`
   - パスワード：WordPress 作成時に控えたパスワード

3. **[プラグイン] > [プラグインを追加]** をクリックする

4. 検索窓に `WP Offload Media Lite for Amazon S3` と入力して検索し、**[今すぐインストール]** をクリックする

5. インストール完了後、**[有効化]** をクリックする

6. プラグイン一覧で WP Offload Media Lite の **[Settings]** をクリックする

7. 以下の順に設定を進める：

   | 手順 | 設定内容 |
   |-----|---------|
   | 1. Select Provider | `Amazon S3` を選択 |
   | 2. Connection Method | `My server is on Amazon Web Services and I'd like to use IAM Roles` を選択 → **[Save & Continue]** |
   | 3. New or Existing Bucket? | `Use Existing Bucket` を選択 |
   | 4. Select Bucket | `Browse existing buckets` を選択 → 自身の S3 バケットを選択 → **[Save Selected Bucket]** |
   | 5. セキュリティ設定 | デフォルトのまま `Keep Bucket Security As Is` をクリック |

#### 5-2. Systems Manager を利用した CloudWatch エージェントの設定と起動

#### 6-1. ネットワークレイヤーを記述する
